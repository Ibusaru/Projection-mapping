## 1. プロジェクト概要

学校の授業におけるプロジェクションマッピング制作プロジェクト向けの、ウェブアプリケーション開発仕様書。

Blenderで作成した3Dモデルをプロジェクターで投影し、来場者がスマートフォンからリアルタイムでエフェクトを操作できる体験を実現する。

| 項目 | 内容 |
|------|------|
| プロジェクト名 | プロジェクションマッピング ウェブ操作システム |
| 制作体制 | 4人班（学校授業） |
| 3Dモデリング | Blender（glTF / GLB 形式でエクスポート） |
| Web表示技術 | Three.js（WebGL） |
| 通信方式 | WebSocket（リアルタイム双方向通信） |
| ユーザーアクセス | QRコードによるスマートフォンアクセス |
| 発注先 | Antigravity |

## 2. システム構成

### 2-1. 全体アーキテクチャ

```
[スマートフォン]
  来場者がQRコードを読み取り
  コントローラー画面にアクセス
       │
       │ WebSocket
       ▼
[WebSocketサーバー]
  Node.js（ws / socket.io）
  操作データを中継・ブロードキャスト
       │
       │ WebSocket
       ▼
[プロジェクター表示画面]
  Three.js で Blender モデルを描画
  受信データをリアルタイムにエフェクト反映
       │
       ▼
  📽️ プロジェクター投影
```

### 2-2. データフロー

1. 来場者がスマートフォンでQRコードを読み取り、コントローラー画面にアクセス
2. 来場者がエフェクト操作を行うと、操作データがWebSocketサーバーへ送信される
3. WebSocketサーバーがプロジェクター表示画面へデータをブロードキャスト
4. Three.jsのレンダリングにリアルタイムで反映される（**目標遅延：100ms以内**）

### 2-3. QRコードアクセス

- URLは公開済みのホスティング環境（Vercel / Netlify 等）を使用
- qrcode.js 等のライブラリでQRコード生成し、印刷用PDFとして出力できること
- スマートフォンブラウザ（iOS Safari / Android Chrome）での動作を保証すること

## 4. 技術仕様

### 4-1. フロントエンド

| 項目 | 内容 |
|------|------|
| 3Dレンダリング | Three.js（最新安定版） |
| GLTFローダー | Three.js 付属の GLTFLoader |
| WebSocket | ブラウザ標準の WebSocket API |
| UIフレームワーク | 指定なし（vanilla JS / React どちらでも可） |
| スタイリング | タッチUI対応、ボタンサイズ44px以上 |

### 4-2. バックエンド（WebSocketサーバー）

| 項目 | 内容 |
|------|------|
| ランタイム | Node.js 20 LTS 以上 |
| ライブラリ | ws または socket.io（実装者に一任） |
| ホスティング | Railway / Render 等（WebSocket対応のもの） |
| メッセージ形式 | JSON（下記仕様に従う） |

### 4-3. WebSocket メッセージ仕様

コントローラーから送信するJSONの基本形式：

```json
{
  "type": "effect",
  "payload": {
    "effectId": "fire",
    "color": "#FF4400",
    "intensity": 0.8
  }
}
```

| フィールド | 説明 |
|------------|------|
| `type` | メッセージ種別。`"effect"` / `"animation"` / `"reset"` |
| `payload.effectId` | エフェクトの識別子（文字列）。プロジェクター側で定義済みのIDを使用 |
| `payload.color` | HEX形式の色コード（例：`#FF4400`） |
| `payload.intensity` | 輝度・強度（`0.0`〜`1.0` の浮動小数点数） |

### 4-4. 3Dモデル連携

- BlenderからglTF 2.0（`.glb`）形式でエクスポートすること
- ファイルサイズは50MB以内を目標とし、テクスチャは適切に圧縮すること
- Three.jsの`GLTFLoader`でロードし、シーンに配置する
- マテリアルはThree.js側でWebSocketデータに応じて動的に変更可能な設計とすること

## 5. 非機能要件

| 項目 | 要件 |
|------|------|
| レスポンス遅延 | 操作からエフェクト反映まで100ms以内（同一LAN環境） |
| 対応ブラウザ | iOS Safari 16以上 / Android Chrome 110以上 / デスクトップChrome |
| 同時接続数 | 最低10台のスマートフォンから同時接続できること |
| フレームレート | プロジェクター表示は30fps以上を維持すること |
| オフライン考慮 | 接続断時はコントローラー画面に再接続中の旨を表示すること |
